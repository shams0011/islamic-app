# AUDIO_ENGINE.md — কুরআন অডিও ইঞ্জিন (P0.6, v163)

## আর্কিটেকচার

তিনটি পৃথক স্তর — প্লেব্যাক-ইঞ্জিনে কোনো UI-লজিক নেই:

| স্তর | দায়িত্ব |
|---|---|
| **`AE`** (ইঞ্জিন) | প্লেব্যাক, কিউ, রিপিট, গতি, গ্যাপ, প্রিলোড, Media Session, রিডার-সিঙ্ক; ভিউ-স্তর `AE.onchange`/`_notify()`-এ সাবস্ক্রাইব করে |
| **`ADL`** (ডাউনলোড ম্যানেজার) | Cache API বাকেট `quran-audio-v1` — সূরা-ভিত্তিক ডাউনলোড, বিরতি/চালু/বাতিল, স্ট্যাটাস, স্টোরেজ-ব্যবহার, ডিলিট, ইন্টেগ্রিটি-চেক (HTTP-OK + audio-type + ≥1KB) |
| **অডিও-শিট** (ভিউ) | `renderAudioSheet()` — ইঞ্জিন-স্টেট থেকে আঁকে, ইঞ্জিনকে কমান্ড পাঠায় |

সেটিংস `RS` স্কিমার `audio` সেকশনে (reciter, speed, autoNext, gapAyah, gapRepeat) — একক ডেটা-লেয়ার নীতি বজায়।

## সক্ষমতা

- **Media Session:** লক-স্ক্রিন/নোটিফিকেশন/ব্লুটুথ/হেডসেট — play, pause, prev/next আয়াত (track-অ্যাকশন), prev/next সূরা (seek-অ্যাকশনে ম্যাপড); প্রতি আয়াতে মেটাডেটা আপডেট (সূরার বাংলা নাম, আয়াত-নম্বর, ক্বারী, আর্টওয়ার্ক)
- **কিউ:** পূর্ণ সূরা (যেকোনো আয়াত থেকে), কাস্টম রেঞ্জ, **পূর্ণ জুয** (একাধিক সূরা জুড়ে), প্রিভিউ (idx/len শিটে)
- **রিপিট:** অফ / আয়াত / N-বার / রেঞ্জ / সূরা / চলমান-পর্যন্ত — রিপিটের মাঝে আলাদা গ্যাপ
- **গতি:** ০.৫–২x, সংরক্ষিত · **আয়াত-গ্যাপ:** ০+ সেকেন্ড
- **সিঙ্ক:** চলমান আয়াতে সোনালি রিং (dematerialize-রিমেটেরিয়ালাইজেও টেকে), অটো-স্ক্রল (সেটিং-চালিত, center-smooth), প্রতি-আয়াত `audio` কাউন্ট RS-স্টেটে — **পুরো পেজ কখনো রি-রেন্ডার হয় না**
- **অফলাইন:** ক্যাশ-হিটে blob-URL প্লেব্যাক; ডাউনলোডে আগে-নামানো ফাইল স্কিপ (ডুপ্লিকেট-মুক্ত + রিজিউমেবল); ৫ ক্বারী
- **ব্যাক-কম্প্যাট:** পুরনো `toggleSurahAudio`/`playVerseAudio` এখন ইঞ্জিন-র‍্যাপার — সব পুরনো বাটন অক্ষত; তাসবিহ-অডিওর সাথে মিডিয়া-এলিমেন্ট ভাগাভাগি দ্বন্দ্বমুক্ত (পরস্পরকে stop করে)

## ভবিষ্যৎ-সামঞ্জস্য

কিউ-আইটেম `{s, a}` স্বয়ংসম্পূর্ণ এবং রিপিট/গ্যাপ/স্পিড কম্পোজেবল — AI উচ্চারণ-ট্রেইনার, মুখস্থ-মোড (রেঞ্জ+N-রিপিট+গ্যাপ ইতিমধ্যেই আছে), শিক্ষক/পরীক্ষা-মোড ও ক্লাউড-সিঙ্ক ইঞ্জিন-API না ভেঙে বসবে।
